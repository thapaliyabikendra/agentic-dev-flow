## Basic SELECT with Filters

**Pattern:** Get active records with standard filters

```sql
SELECT
  e."Id",
  e."SystemName",
  e."DisplayName",
  e."CreationTime"
FROM "Applications" e
WHERE e."TenantId" = @tenantId
  AND e."IsDeleted" = false
  AND e."IsActive" = true
ORDER BY e."CreationTime" DESC
LIMIT @maxResultCount OFFSET @skipCount;
```

**Auto-applied filters:**
- `TenantId = @tenantId` (if multi-tenant)
- `IsDeleted = false` (if soft delete)
- `IsActive = true` (commonly applicable)

## Multi-Entity Joins

**Pattern:** Query with multiple related entities

```sql
SELECT
  awi."Id",
  awi."TaskNumber",
  awi."WorkflowStageDate",
  app."Name" as "ApplicationName",
  task."Name" as "TaskName",
  stage."DisplayName" as "StageName"
FROM "ApplicationWorkflowInstances" awi
JOIN "Applications" app ON app."Id" = awi."ApplicationWorkflowConfigurationId"
  AND app."TenantId" = awi."TenantId"
JOIN "WorkflowStages" stage ON stage."Id" = awi."StageId"
  AND stage."TenantId" = awi."TenantId"
LEFT JOIN "WorkflowSubStages" substage ON substage."Id" = awi."SubStageId"
WHERE awi."TenantId" = @tenantId
  AND awi."IsDeleted" = false
  AND stage."IsActive" = true
ORDER BY awi."WorkflowStageDate" DESC;
```

**Key points:**
- Always join with TenantId match: `AND table."TenantId" = parent."TenantId"`
- LEFT JOIN for optional relationships
- Table aliases: 3-4 letter abbreviations (awi, app, stage)

## Many-to-Many Relationships

**Pattern:** Query through join entities with extra columns

```sql
-- Find operations for user's account types
SELECT DISTINCT
  o."Id",
  o."SystemName",
  o."DisplayName",
  ovat."ExecutionOrder"
FROM "Operations" o
JOIN "OperationUserAccountTypes" ovat
  ON ovat."OperationId" = o."Id"
  AND ovat."TenantId" = o."TenantId"
JOIN "UserAccountTypes" uat
  ON uat."Id" = ovat."UserAccountTypeId"
  AND uat."TenantId" = ovat."TenantId"
WHERE uat."Id" = @userAccountTypeId
  AND o."TenantId" = @tenantId
  AND o."IsDeleted" = false
ORDER BY ovat."ExecutionOrder";
```

**When to use:** Many-to-many relationships always have an explicit join entity table.

## Recursive Hierarchies

**Pattern:** Self-referencing entities (ParentId)

```sql
WITH RECURSIVE category_tree AS (
  -- Anchor: root nodes (no parent)
  SELECT
    c."Id",
    c."ParentId",
    c."Name",
    c."Code",
    1 as "Level"
  FROM "Categories" c
  WHERE c."ParentId" IS NULL
    AND c."TenantId" = @tenantId
    AND c."IsDeleted" = false

  UNION ALL

  -- Recursive: children
  SELECT
    child."Id",
    child."ParentId",
    child."Name",
    child."Code",
    tree."Level" + 1
  FROM "Categories" child
  INNER JOIN category_tree tree ON child."ParentId" = tree."Id"
  WHERE child."TenantId" = @tenantId
    AND child."IsDeleted" = false
)
SELECT * FROM category_tree
ORDER BY "Level", "Code";
```

**Entities that benefit:** Categories, OrganizationUnits, MenuItems, WorkflowConfigurations.

## JSONB Queries

**Pattern:** Query JSONB fields with containment operators

```sql
-- Check if JSONB contains key-value pair
SELECT *
FROM "DynamicForms"
WHERE "FormData" @> '{"required": true}'
  AND "TenantId" = @tenantId
  AND "IsDeleted" = false;

-- Extract value from JSONB
SELECT
  "Id",
  "JsonSchema"->>'version' as schema_version,
  "JsonSchema"->'properties' as properties
FROM "DynamicForms"
WHERE "TenantId" = @tenantId;

-- Query JSONB array contains element
SELECT *
FROM "OperationJobs"
WHERE "OperationParameters" @> '{"tags": ["priority"]}';

-- Check if JSONB has key
SELECT *
FROM "DynamicTables"
WHERE "TableSchema" ? 'columns';
```

**JSONB Operators:**
- `@>` - contains
- `?` - key exists
- `?|` - any key exists
- `?&` - all keys exist
- `->` - get JSON object
- `->>` - get text value
- `#>` - get nested JSON
- `#>>` - get nested text

## Window Functions

**Pattern:** Row numbering, ranking, aggregates per partition

```sql
-- Get latest workflow instance per operation
SELECT DISTINCT ON ("OperationId")
  "OperationId",
  "RequestNumber",
  "WorkflowStageId",
  "WorkflowStageDate",
  FIRST_VALUE("Status") OVER (
    PARTITION BY "OperationId"
    ORDER BY "WorkflowStageDate" DESC
  ) as "LatestStatus"
FROM "OperationWorkflowInstances"
WHERE "TenantId" = @tenantId
  AND "IsDeleted" = false;

-- Row number per operation (for top N per group)
SELECT
  "OperationId",
  "RequestNumber",
  "WorkflowStageDate",
  ROW_NUMBER() OVER (
    PARTITION BY "OperationId"
    ORDER BY "WorkflowStageDate" DESC
  ) as stage_sequence
FROM "OperationWorkflowInstances"
WHERE "TenantId" = @tenantId
  AND "IsDeleted" = false;
```

**Common window functions:**
- `ROW_NUMBER()` - sequential numbers per partition
- `RANK()` / `DENSE_RANK()` - ranking with/without gaps
- `FIRST_VALUE()` / `LAST_VALUE()` - extract values
- `LAG()` / `LEAD()` - previous/next row values
- `SUM() OVER (PARTITION BY)` - running totals

## Aggregations

**Pattern:** Grouping with HAVING

```sql
-- Applications with specific business rules
SELECT
  app."Id",
  app."SystemName",
  array_agg(DISTINCT br."SystemName") as "BusinessRules"
FROM "Applications" app
JOIN "ApplicationTasks" at ON at."ApplicationId" = app."Id"
JOIN "ApplicationTaskRules" atr ON atr."ApplicationTaskId" = at."Id"
JOIN "BusinessRules" br ON br."Id" = atr."BusinessRuleId"
WHERE app."TenantId" = @tenantId
  AND app."IsDeleted" = false
  AND br."SystemName" IN ('Rule1', 'Rule2', 'Rule3')
GROUP BY app."Id", app."SystemName"
HAVING COUNT(DISTINCT br."Id") = 3;  -- All 3 rules matched
```

**Aggregate functions:**
- `COUNT(*)` - row count
- `SUM(column)` - total
- `AVG(column)` - average
- `MIN()` / `MAX()` - extremes
- `ARRAY_AGG(DISTINCT col)` - collect distinct values
- `STRING_AGG(DISTINCT col, ',')` - concatenate

## Full-Text Search

**Pattern:** Case-insensitive pattern matching

```sql
-- Search multiple text columns
SELECT *
FROM "Applications"
WHERE "TenantId" = @tenantId
  AND "IsDeleted" = false
  AND (
    "SystemName" ILIKE '%search%'
    OR "DisplayName" ILIKE '%search%'
    OR "Description" ILIKE '%search%'
  );

-- Prefix search
SELECT *
FROM "Users"
WHERE "UserName" ILIKE 'john%'
  AND "TenantId" = @tenantId;

-- With pagination
SELECT
  COUNT(*) OVER() as "TotalCount",
  u.*
FROM "Users" u
WHERE "UserName" ILIKE '%' || @search || '%'
LIMIT @maxCount OFFSET @skip;
```

**Performance tip:** Add GIN/GIST indexes for full-text search on frequently searched columns:
```sql
CREATE INDEX CONCURRENTLY "IX_Applications_Search" ON "Applications" USING GIN ("SystemName" gin_trgm_ops);
-- Requires pg_trgm extension
```

## Pagination with Count

**Pattern:** Single query returning total count and page results

```sql
WITH filtered AS (
  SELECT awi.*
  FROM "ApplicationWorkflowInstances" awi
  WHERE awi."TenantId" = @tenantId
    AND awi."IsDeleted" = false
    AND awi."IsActive" = true
    AND awi."StageId" = @stageId
)
SELECT
  (SELECT COUNT(*) FROM filtered) as "TotalCount",
  f."Id",
  f."TaskNumber",
  f."WorkflowStageDate",
  f."StageId"
FROM filtered f
ORDER BY f."WorkflowStageDate" DESC
LIMIT @maxResultCount OFFSET @skipCount;
```

**Alternative:** Two separate queries (count + select) is also common.

## Permission Queries

**Pattern:** Complex multi-join permission checks

```sql
-- Get operations accessible to user's account types
SELECT DISTINCT
  o."Id",
  o."SystemName",
  o."DisplayName"
FROM "Operations" o
JOIN "OperationUserAccountTypes" ovat
  ON ovat."OperationId" = o."Id"
  AND ovat."TenantId" = o."TenantId"
JOIN "UserAccountTypes" uat
  ON uat."Id" = ovat."UserAccountTypeId"
  AND uat."TenantId" = ovat."TenantId"
WHERE ovat."UserAccountTypeId" IN (
  SELECT "UserAccountTypeId"
  FROM "AbpIdentity"."UserRoles" ur
  JOIN "AbpIdentity"."Roles" r ON r."Id" = ur."RoleId"
  WHERE ur."UserId" = @userId
  AND r."IsActive" = true
)
  AND o."TenantId" = @tenantId
  AND o."IsDeleted" = false
  AND o."IsActive" = true;
```

**Key tables:**
- `AbpIdentity.Users`
- `AbpIdentity.UserRoles`
- `AbpIdentity.Roles`
- Custom: `UserAccountTypes`, `OperationUserAccountTypes`

## Conditional Updates

**Pattern:** Update only if record exists and meets criteria

```sql
UPDATE "OperationWorkflowInstances"
SET
  "WorkflowStageId" = @newStageId,
  "LastModificationTime" = NOW(),
  "ModifierId" = @userId
WHERE "Id" = @id
  AND "TenantId" = @tenantId
  AND "IsDeleted" = false
  AND "IsActive" = true
RETURNING *;
```

**Important:**
- Always include TenantId and IsDeleted in WHERE
- Use `RETURNING *` to get updated record
- Check affected row count

## Upsert Pattern (INSERT ... ON CONFLICT)

**Pattern:** Insert or update based on natural key

```sql
INSERT INTO "Applications" ("Id", "TenantId", "SystemName", "DisplayName", "IsActive")
VALUES (@id, @tenantId, @systemName, @displayName, true)
ON CONFLICT ("TenantId", "SystemName")
WHERE "IsDeleted" = false
DO UPDATE SET
  "DisplayName" = EXCLUDED."DisplayName",
  "LastModificationTime" = NOW();
```

**Natural key constraint required:**
```sql
CREATE UNIQUE INDEX "IX_Applications_Tenant_SystemName"
ON "Applications" ("TenantId", "SystemName")
WHERE "IsDeleted" = false;
```

## Bulk Operations

**Pattern:** Insert/Update multiple rows efficiently

```sql
-- Batch insert
INSERT INTO "DynamicFormRules" ("Id", "DynamicFormId", "BusinessRuleId", "ExecutionOrder", "TenantId")
VALUES
  (@id1, @formId1, @ruleId1, 100, @tenantId),
  (@id2, @formId2, @ruleId2, 200, @tenantId),
  (@id3, @formId3, @ruleId3, 300, @tenantId);

-- Batch update with CASE
UPDATE "DynamicTableRules"
SET "ExecutionOrder" = CASE "Id"
  WHEN @id1 THEN 100
  WHEN @id2 THEN 200
  WHEN @id3 THEN 300
  ELSE "ExecutionOrder"
END,
"LastModificationTime" = NOW()
WHERE "Id" IN (@id1, @id2, @id3)
  AND "TenantId" = @tenantId;
```

**For large batches (1000+ rows):**
- Use temporary tables
- Consider `COPY` for bulk insert
- Batch in chunks of 100-500 rows

## Date/Time Queries

**Pattern:** TIMESTAMPTZ with timezone awareness

```sql
-- Filter by date range (UTC)
SELECT *
FROM "OperationWorkflowInstances"
WHERE "WorkflowStageDate" >= '2024-01-01 00:00:00+00'
  AND "WorkflowStageDate" < '2024-02-01 00:00:00+00';

-- Group by day (timezone-aware)
SELECT
  date_trunc('day', "CreationTime") as "Day",
  count(*) as "Count"
FROM "ApplicationWorkflowInstances"
WHERE "TenantId" = @tenantId
  AND "IsDeleted" = false
GROUP BY date_trunc('day', "CreationTime")
ORDER BY "Day" DESC;

-- Extract date parts
SELECT
  EXTRACT(YEAR FROM "CreationTime") as "Year",
  EXTRACT(MONTH FROM "CreationTime") as "Month",
  COUNT(*) as "Count"
FROM "Applications"
GROUP BY EXTRACT(YEAR FROM "CreationTime"), EXTRACT(MONTH FROM "CreationTime");

-- Timezone conversion
SELECT
  "CreationTime" AT TIME ZONE 'UTC' as "UtcTime",
  "CreationTime" AT TIME ZONE 'Asia/Kathmandu' as "LocalTime"
FROM "OperationWorkflowInstances";
```

## Webhook & Integration Queries

**Pattern:** Get active webhook subscriptions and retry attempts

```sql
-- Active webhooks for event
SELECT
  ws."Id",
  ws."ConsumerId",
  ac."SystemName" as "ApiClientName",
  we."EventName",
  ws."RetryCount",
  ws."MaxRetryCount"
FROM "WebhookSubscriptions" ws
JOIN "ApiClients" ac ON ac."Id" = ws."ApiClientId"
JOIN "WebhookEvents" we ON we."Id" = ws."WebhookEventId"
WHERE ws."TenantId" = @tenantId
  AND ws."IsDeleted" = false
  AND ws."IsActive" = true
  AND we."EventName" = @eventName
  AND ws."RetryCount" < ws."MaxRetryCount";

-- Failed webhooks needing retry
SELECT
  wl."Id",
  wl."WebhookSubscriptionId",
  wl."AttemptedAt",
  wl."ErrorMessage",
  ac."SystemName" as "ClientName"
FROM "WebhookLogs" wl
JOIN "WebhookSubscriptions" ws ON ws."Id" = wl."WebhookSubscriptionId"
JOIN "ApiClients" ac ON ac."Id" = ws."ApiClientId"
WHERE wl."Succeeded" = false
  AND wl."RetryCount" < wl."MaxRetryCount"
  AND wl."AttemptedAt" > NOW() - INTERVAL '1 hour'
ORDER BY wl."AttemptedAt" ASC;
```

## Union Queries

**Pattern:** Combine results from multiple tables or queries

```sql
-- Union of workflow stages and substages
SELECT
  ws."Id",
  ws."SystemName",
  ws."DisplayName",
  ws."ExecutionOrder",
  'Stage' as "Type"
FROM "WorkflowStages" ws
WHERE ws."TenantId" = @tenantId
  AND ws."IsActive" = true

UNION ALL

SELECT
  wss."Id",
  wss."SystemName",
  wss."DisplayName",
  wss."ExecutionOrder",
  'SubStage' as "Type"
FROM "WorkflowSubStages" wss
WHERE wss."TenantId" = @tenantId
  AND wss."IsActive" = true

ORDER BY "ExecutionOrder";
```

## Performance Patterns

### CTEs for Complex Queries
```sql
WITH active_apps AS (
  SELECT * FROM "Applications"
  WHERE "TenantId" = @tenantId
    AND "IsActive" = true
    AND "IsDeleted" = false
)
SELECT a.*, COUNT(t."Id") as "TaskCount"
FROM active_apps a
LEFT JOIN "ApplicationTasks" t ON t."ApplicationId" = a."Id"
GROUP BY a."Id"
HAVING COUNT(t."Id") > 0;
```

### LATERAL Joins (PostgreSQL-specific)
```sql
-- Get latest workflow instance per application
SELECT
  app.*,
  latest_owi."WorkflowStageDate"
FROM "Applications" app
LEFT JOIN LATERAL (
  SELECT "WorkflowStageDate"
  FROM "OperationWorkflowInstances" owii
  WHERE owii."TenantId" = app."TenantId"
    AND owii."IsDeleted" = false
  ORDER BY owii."WorkflowStageDate" DESC
  LIMIT 1
) latest_owi ON true;
```

### Partial Indexes for Performance
```sql
-- Index for active, non-deleted records
CREATE INDEX CONCURRENTLY "IX_Applications_Active"
ON "Applications" ("SystemName")
WHERE "IsDeleted" = false AND "IsActive" = true;

-- Composite index for common query pattern
CREATE INDEX CONCURRENTLY "IX_OWI_Stage_Date"
ON "OperationWorkflowInstances" ("WorkflowStageId", "WorkflowStageDate" DESC)
WHERE "IsDeleted" = false;
```
