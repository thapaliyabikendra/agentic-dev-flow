# EF Core Configuration

ABP uses EF Core configured via `ModelBuilder` extension methods. `solution-inspector` verified `ApplyConfigurationsFromAssembly` wiring exists. The skill generates one ModelBuilder extension per module/feature.

## File Layout — ModelBuilder Extension Pattern (CRC-E1)

Do NOT use `IEntityTypeConfiguration<T>`. Create a static extension method per module:

- Path: `<Ns>.EntityFrameworkCore/<Feature>/<Feature>EntityFrameworkCoreExtensions.cs`
- Namespace: `<Ns>.EntityFrameworkCore.<Feature>`

```csharp
namespace <Ns>.EntityFrameworkCore.<Feature>;

public static class <Feature>EntityFrameworkCoreExtensions
{
    public static void Configure<Feature>(this ModelBuilder builder)
    {
        builder.Entity<<Entity>>(b =>
        {
            b.ToTable(<Ns>DbProperties.DbTablePrefix + "<EntityPlural>", <Ns>DbProperties.DbSchema);
            b.ConfigureByConvention();

            // Property configurations
            b.Property(x => x.Name)
                .HasMaxLength(<Feature>Constants.FieldLengths.NameMax)
                .IsRequired();

            b.Property(x => x.Amount)
                .HasPrecision(18, 4);

            // Value Object ownership
            b.OwnsOne(x => x.Address, address =>
            {
                address.Property(a => a.Line1)
                    .HasMaxLength(<Feature>Constants.FieldLengths.AddressLine1Max)
                    .IsRequired();
            });

            // Relationships
            b.HasMany<OrderLine>("_lines")
                .WithOne()
                .HasForeignKey("OrderId")
                .OnDelete(DeleteBehavior.Cascade);

            // Indexes — NO explicit names (CRC-E3)
            b.HasIndex(x => x.Name);
            b.HasIndex(x => new { x.Email }).IsUnique();
        });
    }
}
```

Register in DbContext:
```csharp
protected override void OnModelCreating(ModelBuilder builder)
{
    base.OnModelCreating(builder);
    builder.Configure<Feature>();
}
```

## Configure Method Order

1. `ToTable(prefix + plural, schema)`
2. `ConfigureByConvention()` — ABP configures audit fields, soft delete, tenant id, concurrency stamp automatically
3. Property configurations
4. Value Object ownership (`OwnsOne`)
5. Relationships
6. Indexes

## Property Configuration

| FS declares | EF configuration |
|---|---|
| `maxLength: 128` | `.HasMaxLength(<Feature>Constants.FieldLengths.<Field>Max)` |
| `required: true` | `.IsRequired()` |
| `precision: 18, scale: 4` | `.HasPrecision(18, 4)` |
| `column: "CustomColumn"` | `.HasColumnName("CustomColumn")` |
| `stored as string` | `.HasConversion<string>()` for enums |

Never inline numbers for lengths — use `<Feature>Constants.FieldLengths.*`.

## Critical Rules from Code Review Checklist

**CRC-E2: No Explicit Soft-Delete Query Filters**
Do NOT add `HasQueryFilter(x => !x.IsDeleted)`. ABP handles this via `ISoftDelete` + `ConfigureByConvention()`.

**CRC-E3: No Explicit Index Names**
Do NOT use `.HasName("IX_...")`. Let EF Core auto-name indexes.

**CRC-E4: TenantId Is Always Nullable**
Do NOT configure `b.Property(x => x.TenantId).IsRequired(false)`. TenantId is nullable by default. No explicit config needed.

## Enum Storage

Default: enums stored as `int` in database. If CLAUDE.md `enum_serialization` says `camelCase strings, global`, that applies to **JSON boundary only** — database still stores `int` unless Entity page explicitly declares string storage.

## Relationships

Cross-aggregate: `HasOne<T>().WithMany().HasForeignKey(x => x.ForeignId).OnDelete(Restrict)`. Never cascade across aggregate boundaries. Referenced aggregates use FK property only, not navigation.

## Migrations

**Never generated by this skill.** After completion, user runs:
```
dotnet ef migrations add Add<Feature>Tables --context <Ns>DbContext --output-dir Migrations --project <Ns>.EntityFrameworkCore --startup-project <Ns>.HttpApi.Host
```

## EF Configurations NEVER Contain

Business logic, seed data, `DbSet<T>` properties, `OnModelCreating` overrides, explicit soft-delete filters, explicit index names, TenantId IsRequired configuration.
