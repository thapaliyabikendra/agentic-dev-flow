# IRS-to-GSD Artifact Mapping

Rules for mapping Implementation Readiness Spec (IRS) content to GSD workflow artifacts.

---

## Mapping Table

| IRS Section | GSD Artifact | How It Maps |
|-------------|-------------|-------------|
| Section 8: Epics | GSD Phases in `.planning/ROADMAP.md` | Each epic becomes one GSD phase |
| Section 8: Stories | GSD Plans (PLAN.md) within a phase | Each story or small story group becomes one plan |
| Section 9: Sprint Breakdown | Phase ordering in ROADMAP.md | Sprint sequence defines phase dependency chain |
| Section 3: Aggregates | Phase CONTEXT.md "Implementation Decisions" | Aggregate invariants and commands lock design |
| Section 4: BDS Scenarios | Plan verify blocks | BDS scenarios become verification criteria |
| Section 5: Integration Contracts | Phase RESEARCH.md questions | Contracts drive research on integration approach |
| Section 6: Key Decisions | Phase CONTEXT.md "Constraints" | DDR constraints inject into context |
| Section 7: Open Questions | Phase prerequisites | Blocking questions must resolve before phase planning |

---

## REQUIREMENTS.md Template

Write to `.planning/REQUIREMENTS.md`. This format is compatible with `gsd:new-milestone`.

```markdown
# Requirements: {Feature Name}

Source: docs/features/{feature-slug}/FEATURE_SPEC.md
Generated: {date}

## Overview

{From IRS Section 1 — Business Summary, 2-3 paragraphs}

## Requirement Groups

### RG-01: {Epic 1 Title}

**Scope:** {From IRS Epic 1 scope}
**Priority:** {High / Medium / Low — based on dependency order}
**Depends On:** {Other RG IDs or "None"}

#### Requirements

- **REQ-01.01:** {Story 1.1 title}
  - Acceptance: {BDS scenario refs}
  - Aggregates: {list}
  - Commands: {list}

- **REQ-01.02:** {Story 1.2 title}
  - Acceptance: {BDS scenario refs}
  - Aggregates: {list}
  - Commands: {list}

### RG-02: {Epic 2 Title}
...

## Open Questions

{From IRS Section 7 — only blocking-implementation and blocking-design items}

1. {Question} — blocks: {RG-ID or REQ-ID}

## Constraints

{From IRS Section 6 — active DDR/ADR constraints}

- {CONST-ID}: {constraint statement} (source: {DDR/ADR-NNN})

## Source Documents

{From IRS Section 10 — DDD Document Index}
```

## Phase CONTEXT.md Template

Write to `.planning/phases/{NN}-{epic-slug}/{NN}-CONTEXT.md`. One per epic.

```markdown
# Phase Context: {Epic Title}

Source Epic: {IRS Section 8 epic reference}
Feature Spec: docs/features/{feature-slug}/FEATURE_SPEC.md

## Goal

{Epic scope description from IRS}

## Implementation Decisions

{From IRS Section 3 — aggregates involved in this epic}

### {Aggregate Name}

- Root Entity: {name}
- Key Invariants: {INV-ID list with descriptions}
- Commands in scope: {list}
- Events raised: {list}
- Lifecycle states: {list}

## Integration Points

{From IRS Section 5 — contracts relevant to this epic}

- {Contract}: {type} with {context} — {status}

## Constraints

{From IRS Section 6 — DDRs/ADRs that constrain this epic}

- {DDR/ADR-NNN}: {constraint statement}

## Acceptance Criteria

{From IRS Section 4 — BDS scenario summaries for commands in this epic}

### {CommandName}
- Happy path: {scenario title}
- Key failure: {scenario title}
- Idempotency: {scenario title}

## Open Questions

{From IRS Section 7 — questions relevant to this epic only}
```

## Workflow: IRS to GSD Phases

1. **Read IRS** — Parse Section 8 epics to determine phase count and ordering
2. **Write REQUIREMENTS.md** — Generate from full IRS using template above
3. **Determine phase numbering** — Check `.planning/ROADMAP.md` for existing phase numbers. New phases continue the sequence.
4. **Create phase directories** — `mkdir .planning/phases/{NN}-{epic-slug}/`
5. **Write CONTEXT.md** — One per phase, using template above
6. **Do NOT write PLAN.md** — Plans are generated by `gsd:plan-phase` which reads CONTEXT.md
7. **Do NOT write RESEARCH.md** — Research is generated by `gsd:research-phase` or `gsd:plan-phase`
8. **Report** — List all created files

---

## Phase Numbering Convention

If existing phases go up to 13, new phases start at 14:
- Phase 14: Epic 1
- Phase 15: Epic 2
- Phase 16: Epic 3

If a specific number range is requested by the user, use that instead.
