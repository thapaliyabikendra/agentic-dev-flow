  # Bank Settings вҖ” LC Issuance Checklist Management

  1. Feature Overview

  Bank administrators manage LC issuance verification checklists through a
  configurable, tenant-scoped item list. Five core operations (create, read,
  update, delete, reorder) enable flexible control over which requirements are
  enforced during LC review by Branch and CTF staff. Items are versioned by
  sequence number and toggled between active/inactive states without deletion.
  All operations are role-gated via TradeFinancePermissions.BankSettings.*.

  Module Initials: BS
  Aggregate Root: LcIssuanceChecklistItem
  Tenancy Model: Per-customer (TenantId scoping required)
  Bounded Context: Bank Settings

  ---
  2. Open Blockers

  CONFLICT-07 (HIGH вҖ” BLOCKING)
  Per-tenant data scoping ambiguity. FRS documents do not explicitly state that
  checklist items are scoped to the authenticated tenant; however, CLAUDE.md
  declares tenancy: per-customer, requiring all mutable entities to be
  tenant-scoped. All Commands and Queries must enforce TenantId filtering.
  Impact: cross-tenant access prevention, audit implications.

  Proposed Resolution: Confirm tenant scoping per CLAUDE.md; implement TenantId:
   Guid property on LcIssuanceChecklistItem; filter all Queries and Commands by
  currentTenant.Id.
